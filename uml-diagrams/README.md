# 慧医数字医疗应用系统 — UML 设计文档

本项目按**软件工程标准流程**，使用 **PlantUML** 绘制完整的 UML 设计图，覆盖需求分析、静态建模与动态建模三大阶段，共 **18 张图**。

所有图源文件为 `.puml`，可用 PlantUML（本地 / VS Code 插件 / 在线 https://www.plantuml.com/plantuml/uml/ ）渲染为 PNG/SVG。

---

## 一、项目背景

| 项 | 内容 |
|---|---|
| 系统名称 | 慧医数字医疗应用系统 |
| 技术栈 | Vue 2 + Spring Boot 2.5.3 + Spring Security + MyBatis + MySQL 8 + Redis 7 |
| 架构 | 前后端分离，Nginx 同源发布并代理 `/api` |
| 角色 | 管理员（ROLE_1，全量维护）、医生（ROLE_2，业务信息只读） |
| 数据来源 | `sql/medical.sql`(17表) + 迁移(3表) + `process-docs/01~02` 需求与设计文档 |

```

│  需求分析        │──▶ 用例图
├─────────────────┤

```

## 四、如何渲染

### 方式 1：在线渲染（最快）
复制 `.puml` 文件内容到 https://www.plantuml.com/plantuml/uml/ 即可。

### 方式 2：VS Code 插件
安装 `PlantUML` 插件，打开 `.puml` 文件后 `Alt+D` 预览，或 `Ctrl+Shift+P` → "PlantUML: Export" 导出 PNG/SVG。

### 方式 3：命令行（需 Java + plantuml.jar）
```bash
# 一次性渲染全部图
java -jar plantuml.jar -charset UTF-8 -o out uml-diagrams/**/*.puml
# 输出到各自目录的 out/ 子文件夹
```

### 方式 4：本仓库已尝试渲染校验
渲染脚本见 `uml-diagrams/render-all.sh`（需本地 `plantuml.jar`）。已用 PlantUML 语法校验通过；如需出图，按方式 1/2/3 任选其一即可。

## 五、关键设计要点（图内提炼）

1. **安全**：Token 以 SHA-256 摘要存 Redis，不存明文；角色取自服务端会话，不信任客户端 `roleName`。
2. **事务边界**：医生新增/删除、密码重置、药品+关系保存均为单事务，失败整体回滚。
3. **引用完整性**：公司/城市被政策引用时拒绝删除（RESTRICT），不静默级联。
4. **上传安全**：2MB 上限、仅 JPEG/PNG、解码校验真实图片、UUID 文件名、可配置目录与基址。
5. **权限矩阵**：管理员全量；医生仅业务 GET，无写入口。
6. **数据质量**：迁移时归档脏关系到 `data_quality_issue` 再清理，保留审计轨迹。
