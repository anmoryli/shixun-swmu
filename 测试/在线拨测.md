# 在线拨测

| 文档项 | 内容 |
|---|---|
| 对象 | 本机或受控演示环境中的慧医数字医疗应用系统 |
| 默认地址 | Web：`http://localhost:9092`；后端：`http://localhost:8082` |
| 默认原则 | 只读优先；除登录/退出产生会话外，不执行任何业务 POST、PUT、DELETE，不新增或修改业务数据 |
| 适用场景 | Docker 更新后冒烟、定时可用性监控、演示前检查、故障恢复确认 |
| 非适用场景 | 完整 57 项回归会创建和清理数据，应在发布测试阶段执行，不作为高频在线拨测 |
| 编码与日志 | UTF-8；日志仅保留探针、时间、状态、耗时和错误分类，不保存密码、Authorization 或完整响应体 |

## 拨测分层

| 层级 | 是否需要凭据 | 是否改动业务数据 | 覆盖内容 | 失败含义 |
|---|---|---|---|---|
| L0 公共可用性 | 否 | 否 | 9092 首页、静态资源、8082 健康接口 | Web/后端进程、端口或依赖不可用 |
| L1 安全边界 | 否 | 否 | 未登录访问仪表盘应返回业务码 `10006` | 认证拦截契约异常 |
| L2 认证只读 | 是，专用只读账号 | 仅创建/删除会话 | 登录、权限、首页、药品分页、图片 URL | 认证、Redis 会话、数据库查询或图片链路异常 |
| L3 发布回归 | 是，专用测试账号 | 会创建并清理测试数据 | 57 项黑盒回归 | 业务发布质量异常；只在发布窗口运行 |

## 探针设计

| 探针编号 | 请求 | 前置条件 | 断言 | 超时 | 建议频率 | 数据副作用 |
|---|---|---|---|---:|---:|---|
| DIAL-WEB-01 | `GET http://localhost:9092/` | Web 容器已启动 | HTTP 200；响应非空；包含 HTML 文档标识 | 3 秒 | 1 分钟 | 无 |
| DIAL-WEB-02 | 解析首页引用的首个 JS/CSS 后执行 `GET` | DIAL-WEB-01 成功 | HTTP 200；响应长度大于 0；不接受 HTML 错误页伪装成静态资源 | 3 秒 | 5 分钟 | 无 |
| DIAL-API-01 | `GET http://localhost:8082/actuator/health` | 后端已启动 | HTTP 200；JSON `status` 为 `UP` | 5 秒 | 1 分钟 | 无 |
| DIAL-AUTH-01 | 无 Token `GET http://localhost:8082/api/dashboard` | 后端可用 | HTTP 200；业务码为 `10006`，证明接口受保护 | 5 秒 | 5 分钟 | 无 |
| DIAL-LOGIN-01 | 表单 `POST http://localhost:8082/api/login` | 注入专用拨测账号 | HTTP 200；业务码 `20000`；`data.token` 非空；不得记录 Token | 5 秒 | 10 分钟 | 创建 Redis 会话，不改业务数据 |
| DIAL-PERM-01 | 带 Token `GET /api/permissions` | DIAL-LOGIN-01 成功 | HTTP 200；业务码 `20000`；权限集合存在 | 5 秒 | 10 分钟 | 无 |
| DIAL-HOME-01 | 带 Token `GET /api/dashboard` | DIAL-LOGIN-01 成功 | HTTP 200；业务码 `20000`；返回首页聚合数据结构 | 5 秒 | 5 分钟 | 无 |
| DIAL-READ-01 | 带 Token `GET /api/drugs/1/10?name=` | DIAL-LOGIN-01 成功 | HTTP 200；业务码 `20000`；分页结构存在；允许结果为空 | 5 秒 | 5 分钟 | 无 |
| DIAL-IMAGE-01 | `GET` 药品记录中的非空图片 URL；无记录时使用已批准的固定图片 URL | 已获得图片 URL | HTTP 200；`Content-Type` 以 `image/` 开头；内容长度大于 0 | 5 秒 | 5 分钟 | 无 |
| DIAL-LOGOUT-01 | 带 Token `POST /api/logout` | 认证拨测结束 | HTTP 200；业务码 `20000`；无论前序是否失败均在 `finally` 中尝试 | 5 秒 | 每次认证拨测后 | 删除本次会话，不改业务数据 |

## 结果与指标字段

| 字段 | 类型 | 示例 | 记录规则 |
|---|---|---|---|
| `timestamp` | ISO 8601 时间 | `2026-07-13T16:00:00+08:00` | 使用 Asia/Shanghai 或同时保存 UTC |
| `probe_id` | 字符串 | `DIAL-API-01` | 必填，保持稳定命名 |
| `target` | 字符串 | `backend-local` | 使用逻辑名称，避免把含密钥的 URL 写入日志 |
| `success` | 布尔 | `true` | HTTP 和业务断言均通过才为真 |
| `http_status` | 整数 | `200` | 无响应时填空并记录错误类 |
| `api_code` | 整数 | `20000` | 仅业务 API 记录；健康和静态资源为空 |
| `duration_ms` | 整数 | `243` | 从请求发出到完整断言结束 |
| `error_class` | 枚举 | `超时`、`连接失败`、`状态异常`、`业务码异常`、`结构异常` | 禁止直接保存可能含敏感信息的完整响应 |
| `release_id` | 字符串 | 提交短哈希或镜像标签 | 用于识别“代码已更新但容器仍是旧页面” |

## 频率、阈值与告警

| 规则编号 | 触发条件 | 等级 | 通知内容 | 恢复条件 |
|---|---|---|---|---|
| ALERT-01 | DIAL-WEB-01 或 DIAL-API-01 连续 3 次失败 | P1 | 环境、首次失败时间、最近状态、错误类、发布标识 | 连续 2 次成功 |
| ALERT-02 | 登录、权限、首页任一探针在最近 5 次中失败 ≥ 3 次 | P1 | 失败探针、HTTP/业务码、耗时、后端健康状态 | 连续 3 次完整链路成功 |
| ALERT-03 | 图片探针连续 3 次失败，但 API 健康 | P2 | 图片 URL 的脱敏路径、状态、内容类型、持久卷状态检查建议 | 连续 2 次成功 |
| ALERT-04 | 任一核心探针单次耗时 > 3,000 ms，或 5 个窗口 P95 > 1,000 ms | P2 | 探针、P50/P95/最大值、窗口范围 | 连续 3 个窗口 P95 ≤ 1,000 ms |
| ALERT-05 | 9092 页面可达但 `release_id`/静态资源摘要与目标版本不一致 | P1 | 目标提交、实际镜像标签/摘要、缓存信息 | 容器重建后版本一致且新旧 UI 冒烟通过 |
| ALERT-06 | 账号未配置、凭据读取失败 | 配置告警 | 仅提示变量或密钥条目缺失，不输出值 | 下一次认证拨测成功读取并完成登录 |

## 凭据与数据安全

| 控制项 | 要求 | 禁止行为 |
|---|---|---|
| 账号 | 使用专用、最小权限、可审计的拨测账号；优先医生只读角色 | 使用个人管理员账号长期拨测 |
| 注入 | 从操作系统环境变量或密钥管理服务读取 `DIAL_USERNAME`、`DIAL_PASSWORD` | 写入脚本、Git、Markdown、命令历史或镜像 |
| Token | 仅保存在进程内存，认证链路结束后调用退出；异常路径也尝试清理 | 输出到控制台、报告、URL 查询参数或本地文件 |
| 响应 | 只提取断言字段；失败时记录错误分类和经过脱敏的摘要 | 保存完整用户、医生、政策或 Token 响应 |
| 网络 | 非本机环境必须使用 HTTPS，并校验证书 | 使用 `-SkipCertificateCheck` 作为长期配置 |
| 写操作 | 默认禁用业务 POST/PUT/DELETE；完整回归在隔离窗口单独执行 | 用在线探针新增城市、公司、医生、药品或上传文件 |
| 图片 | 使用现有图片 URL，只做 GET；固定 URL 必须来自可公开测试数据 | 在线探针反复上传图片造成持久卷膨胀 |

## PowerShell 脚本伪代码

| 步骤 | 伪代码行为 | 关键断言 |
|---:|---|---|
| 1 | 设置 `$webBase='http://localhost:9092'`、`$apiBase='http://localhost:8082'`，从环境读取账号密码 | 地址不包含凭据；缺少凭据时仍执行 L0/L1，并把 L2 标为“未配置” |
| 2 | 请求 `$webBase/` 并解析一个静态资源地址 | 首页 200、HTML 非空、静态资源 200 且非 HTML 错误页 |
| 3 | 请求 `$apiBase/actuator/health` | 200 且 `status='UP'` |
| 4 | 无 Token 请求 `$apiBase/api/dashboard` | 业务码 `10006` |
| 5 | 若凭据存在，向 `$apiBase/api/login` 提交表单；Token 只赋给内存变量 | 业务码 `20000` 且 `data.token` 非空 |
| 6 | 用内存 Header 依次 GET 权限、首页、药品分页 | 各响应业务码 `20000`；结构字段存在 |
| 7 | 从药品响应选第一个非空图片地址，解析相对/绝对 URL 后 GET | 200、图片内容类型、内容非空 |
| 8 | `finally` 中调用退出并立即清空 Token 变量 | 不因退出失败覆盖原始失败；不输出 Token |
| 9 | 写入一行脱敏指标并计算连续失败和延迟窗口 | 告警只含探针、状态、耗时、错误类和发布标识 |

```powershell
# 伪代码：用于说明控制流，不应直接当作生产脚本运行。
$webBase = 'http://localhost:9092'
$apiBase = 'http://localhost:8082'
$token = $null

Invoke-Probe 'DIAL-WEB-01' { Assert-WebHome "$webBase/" -TimeoutSec 3 }
Invoke-Probe 'DIAL-WEB-02' { Assert-FirstStaticAsset "$webBase/" -TimeoutSec 3 }
Invoke-Probe 'DIAL-API-01' { Assert-Health "$apiBase/actuator/health" -Expected 'UP' }
Invoke-Probe 'DIAL-AUTH-01' { Assert-ApiCode "$apiBase/api/dashboard" 10006 }

try {
    if ($env:DIAL_USERNAME -and $env:DIAL_PASSWORD) {
        $login = Invoke-Login "$apiBase/api/login" $env:DIAL_USERNAME $env:DIAL_PASSWORD
        $token = $login.data.token
        $headers = @{ Authorization = $token }
        Assert-ApiCode "$apiBase/api/permissions" 20000 $headers
        Assert-ApiCode "$apiBase/api/dashboard" 20000 $headers
        $drugs = Invoke-ReadOnlyApi "$apiBase/api/drugs/1/10?name=" $headers
        Assert-FirstExistingImage $drugs $apiBase
    }
}
finally {
    if ($token) { Invoke-BestEffortLogout "$apiBase/api/logout" $token }
    $token = $null
}

Write-RedactedMetrics
Evaluate-AlertRules
```
