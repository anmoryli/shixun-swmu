# 移动应用测试

| 文档项 | 内容 |
|---|---|
| 产品形态 | **响应式 Web 应用，不是原生 Android/iOS 应用** |
| 前端技术 | Vue 2、Vue Router Hash 模式、Element UI；浏览器访问 Web 地址 |
| 核心参考视口 | 390 × 844 CSS 像素，竖屏 |
| 重点平台 | Android Chrome、iOS Safari；同时验证可用的当前主版本和上一主版本 |
| 双界面范围 | 经典界面 `classic` 与新界面 `ios26` 都必须覆盖，不能只测默认新界面 |
| 静态基础 | 页面含 viewport；新界面样式包含 1024/768/640/420 px 断点、390 px 参考规则、安全区和减少动态效果 |
| 结果口径 | 下表均为测试设计，未执行项写“待执行”；禁止把响应式代码存在等同于真机通过 |
| 文件编码 | UTF-8 中文 Markdown |

## 适用与不适用范围

| 能力 | 适用性 | 测试处理 |
|---|---|---|
| 移动浏览器页面、触控、横竖屏、软键盘、弱网、可访问性 | 适用 | 纳入本计划并在 Android Chrome、iOS Safari 执行 |
| 新旧 UI 切换及刷新保持 | 适用 | 两种浏览器、两种方向、登录前后均执行 |
| Web 图片上传 | 适用 | 通过浏览器文件选择器选择 JPG/PNG，验证大小和伪造文件边界 |
| APK/IPA 安装、升级、覆盖安装、卸载 | 不适用 | 项目没有 APK/IPA 产物，不计入通过率 |
| Android Manifest、iOS Info.plist、应用签名、商店审核 | 不适用 | 项目不是原生应用，不生成相关缺陷 |
| 原生权限弹窗、推送通知、后台保活、深链、Universal Link | 不适用 | 当前仓库未实现原生容器或相关能力 |
| 原生崩溃、ANR、启动耗时、包体积、电量专项 | 不适用 | 改为关注浏览器页面错误、长任务、资源体积和首屏耗时 |
| PWA 安装、Service Worker 离线缓存 | 不适用 | 当前仓库没有 PWA 清单和 Service Worker；若后续增加则重新纳入 |

## 网络与环境矩阵

| 网络编号 | 模拟条件 | 重点检查 | 通过标准 |
|---|---|---|---|
| N-01 | 稳定 Wi-Fi，无限速 | 功能基线、图片清晰度、控制台错误 | 关键链路全部通过，无未处理脚本异常 |
| N-02 | 弱网：下行约 1.5 Mbps、上行约 750 Kbps、往返延迟约 200 ms | 首屏、首页聚合、药品图片、加载反馈、防重复点击 | 页面有明确反馈；不出现重复提交；10 秒请求超时后可恢复重试 |
| N-03 | 高延迟：往返延迟约 400 ms | 登录按钮、分页、切换路由、弹窗保存反馈 | 操作状态可见，用户不会误认为点击无效 |
| N-04 | 请求中断网 5 秒后恢复 | 登录、首页查询、图片加载 | 失败有可理解提示；恢复后刷新/重试可用；无脏状态 |
| N-05 | 完全离线 | 首次访问、已打开页面再操作 | 不白屏；给出网络错误；不宣称支持离线业务 |
