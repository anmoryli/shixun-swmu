# 慧医数字医疗应用系统测试用例

## 一、用例说明

| 项目 | 内容 |
|---|---|
| 测试对象 | Vue 2 前端、Spring Boot 后端、MySQL、Redis、文件上传链路及 Docker Compose 部署 |
| 需求基线 | process-docs/01-requirements-baseline.md、需求规格说明书及 10 段需求演示视频 |
| 接口基线 | api-tests/run_api_tests.py、process-docs/project-management/blackbox-57-backlog.csv |
| 编号规则 | 接口用例 001 至 057 与现有 57 项黑盒回归同序；补充用例按上传、界面、响应式、容器分类编号 |
| 结果说明 | 本文件描述测试设计，不代表本次已经执行；真实执行状态以“测试执行.md”为准 |
| 数据原则 | 新增数据使用带时间戳的唯一值；删除按药品、政策、材料、医生、销售地点、公司、城市的依赖逆序执行 |
| 安全原则 | 测试记录不得保存密码、数据库密钥、Redis 密钥、Authorization 或完整令牌 |
