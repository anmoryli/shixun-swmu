# 测试质量看板

| 文档项 | 内容 |
|---|---|
| 项目 | 慧医数字医疗应用系统 |

| 看板用途 | 统一发布前质量口径；区分“历史证据”“当前提交实测”“待采集”，禁止用旧报告代替当前版本结果 |

## 状态与统计口径

| 状态 | 含义 | 使用规则 |
|---|---|---|
| 绿色 | 已有可定位证据，且数值达到绿色阈值 | 必须填写证据文件或报告编号 |
| 黄色 | 已有证据但只覆盖部分范围、证据不是当前提交，或数值落入预警区 | 发布前应补测或由负责人书面接受风险 |
| 红色 | 数值触发阻断阈值，或关键门禁失败 | 阻止发布，修复后全量回归 |
| 待采集 | 没有可核对的执行记录，或当前值无法从仓库材料得出 | 不填猜测值，不按“通过”统计 |
| 不适用 | 产品形态确实不包含该能力 | 必须写明原因，不得用于抬高通过率 |

## 发布门禁概览

| 质量域 | 已证实情况 | 尚未闭环 | 当前判断 |
|---|---|---|---|
| 后端功能与权限 | 57/57 黑盒接口回归、8/8 Maven 测试；含登录、Token 失效、管理员/医生权限和八类业务 | 需要在目标发布提交上重跑 | 基线通过，当前提交待确认 |
| 接口性能 | 已有单轮平均 309.9 ms、P95 550 ms、最大 639 ms | 未做并发、持续压测和前端性能采集 | 功能回归场景通过，容量结论不足 |
| 部署与图片 | 历史 Compose 2/2 healthy；测试图片经过重建仍可访问 | 当前前端镜像内容、示例图片和 9092/8082 链路待重新留证 | 有条件通过 |
| 桌面端新旧 UI | 代码已提供全局切换和本地持久化 | 缺登录后关键页面、刷新保持和业务回归的浏览器证据 | 待采集 |
| 移动 Web | 有 viewport、390 px 参考断点、安全区和减少动态效果代码 | 缺 Android Chrome、iOS Safari、横竖屏、弱网和可访问性实测 | 待采集 |
| 在线可用性 | 已有健康接口和部署脚本 | 未配置周期任务、指标存储和告警接收人 | 待采集 |

## 数据采集与更新责任

| 数据 | 建议采集时机 | 最小保留内容 | 更新责任 | 备注 |
|---|---|---|---|---|
| Maven 测试 | 每次后端合并、发布前 | 命令、提交号、通过/失败/错误数、报告路径 | 后端开发或持续集成 | 失败时保留首个失败堆栈 |
| API 黑盒回归 | 每次后端/数据库变更、发布前 | JSON、JUnit XML、Markdown、目标地址、提交号 | 测试人员 | 报告不得保存密码或完整 Token |
| 前端生产构建 | 每次前端变更 | 命令、提交号、退出码、产物摘要 | 前端开发或持续集成 | 历史构建不能代替当前提交 |
| Docker 健康 | 每次镜像重建后 | `compose ps`、镜像标签/摘要、页面版本标识 | 部署人员 | 必须证明容器实际使用新镜像 |
| 新旧 UI 与移动端 | 每次样式、布局、路由变更 | 设备/浏览器、视口、模式、结果、缺陷号、关键截图 | 测试人员 | 经典界面和新界面分开统计 |
| 在线拨测 | 生产或演示环境持续采集 | 时间、探针、状态、耗时、错误类别；不保存密钥 | 运维或值班人员 | 按 `在线拨测.md` 的频率和告警规则执行 |
| 代码扫描 | 每次发布候选版本 | 工具版本、规则集、各等级数量、误报说明 | 开发负责人 | PDF 之外应补一份可检索的摘要表 |

## 看板结论

| 结论项 | 结论 |
|---|---|
| 已达到 | 现有证据支持后端接口功能、基础权限和单轮响应时间达到既定基线 |
| 未达到 | 当前提交构建、当前 Docker 镜像、新旧 UI 浏览器执行、移动端兼容、持续拨测和结构化代码扫描数据尚未形成完整证据链 |
| 发布建议 | 不把历史 57/57 和 2/2 healthy 直接视为当前版本全量放行；补齐当前提交的构建、容器版本、新旧 UI、390×844 移动端和只读拨测记录后再更新最终结论 |
