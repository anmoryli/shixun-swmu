## 5. 接口契约设计

| 设计编号 | 方法与路径 | 身份 | 正常断言 | 负向与异常断言 |
|---|---|---|---|---|
| SJ-API-01 | `POST /api/login` | 公开 | 表单提交可用，兼容 JSON；响应含统一结构、成功码、令牌和用户信息 | 缺参、错误密码、禁用账号、畸形 JSON 不产生会话；字段超过服务端上限时明确拒绝 |
| SJ-API-02 | `GET /api/permissions` | 已登录 | 权限树可被 Vue 动态路由解析；角色来自 Redis 会话 | 无/伪造/过期/退出后令牌返回未登录结果；客户端 `roleName` 不影响响应 |
| SJ-API-03 | `POST /api/logout` | 已登录 | 返回统一成功响应，Redis 会话被删除 | 重复退出和失效令牌行为稳定，不产生服务器错误 |
| SJ-API-04 | `GET /api/dashboard` | 管理员、医生 | 返回八类数量、医生分布和资讯；数值类型稳定 | 空库、局部空集合和数据库异常不得返回结构混乱的成功数据 |
| SJ-API-05 | `GET /api/citys/{pn}/{size}`、`GET /api/citys` | 管理员、医生 | 分页和全量结构与前端约定一致 | 非法页码、超大页长、非法搜索值不导致 500 或无界返回 |
| SJ-API-06 | `/api/companys`、`/api/sales`、`/api/company_policys`、`/api/medical_policys` | 管理员写、双方读 | 各模块查询及管理员增删改返回统一结构，关联对象字段正确 | 医生 POST/PUT/DELETE 一律拒绝；无效关联和被引用删除不落脏数据 |
| SJ-API-07 | `/api/doctors`、`/api/doctors/info`、`PUT /api/doctors/reset/{accountId}` | 管理员写、双方读 | 查询、维护和重置契约与 Vuex 调用一致 | 重复手机号、无效账号、医生越权、事务异常均有明确业务结果 |
| SJ-API-08 | `/api/drugs/{pn}/{size}`、`/api/materials` | 管理员写、双方读 | 分页、搜索、增删改及兼容分页键正确 | 空关联、重复关联、非法编号、医生写入、删除不存在记录行为稳定 |
| SJ-API-09 | `POST /api/base/upload` | 管理员 | `multipart/form-data` 的 `file` 字段返回统一结构和 `/image/{name}` URL | 医生/匿名、空文件、错误 MIME、伪造内容、超过限制均被拒绝且无残留 |
| SJ-API-10 | `GET /image/{name}` | 公开 | 已上传图片可读取，类型和内容正确 | 不存在文件和非法路径不泄漏本地目录，不执行任意文件访问 |
| SJ-API-11 | `GET /actuator/health` | 公开 | 应用、MySQL、Redis 健康状态可用于部署门禁 | 依赖异常时状态非 UP 且拨测可识别，不暴露密码和连接串 |
| SJ-API-12 | 全部接口 | 按接口定义 | 响应统一为 `{code,message,data}`；成功码 `20000`；分页含 `list,total,pages,pageNum,pageSize` 及模块兼容键 | HTTP 状态与业务码不相互矛盾；错误响应无堆栈、SQL、密码、Authorization 或完整 Token |

## 7. 边界值设计

| 设计编号 | 对象 | 有效边界与代表值 | 无效边界与代表值 | 预期原则 |
|---|---|---|---|---|
| SJ-BND-01 | 登录用户名 | 前端长度 3、11；服务端最大 255；使用真实有效账号作正常类 | 空、2、12、255 后一位；仅空白 | 前端按 3～11 校验；绕过前端时服务端仍有 255 上限且不得 500 |
| SJ-BND-02 | 登录密码 | 前端长度 5、18；服务端最大 255 | 空、4、19、256；错误密码 | 客户端校验与服务端防御分别验证；失败不创建 Token |
| SJ-BND-03 | 医生姓名 | 2、4 个中文字符 | 空、1、5、数字、字母、混合字符 | 页面阻止不合规提交；直接接口输入不得造成脏数据或服务器异常 |
| SJ-BND-04 | 医生年龄 | 18、19、99、100 的整数 | 空、17、101、小数、文本、负数 | 页面给出明确提示；服务端异常输入行为可控，数据库不落非法值 |
| SJ-BND-05 | 手机号 | 符合现有中国大陆手机号正则的 11 位号码 | 10/12 位、非法号段、字母、空、已存在手机号 | 格式错误被拒绝；重复手机号返回 `10001`；原记录不变 |
| SJ-BND-06 | 通用名称 | 长度 4、24 | 空、3、25、全空格、超长中文 | 页面规则一致；接口不得因超长或空白返回 500 |
| SJ-BND-07 | 政策/说明文本 | 长度 4、480；中文、中英文混合 | 空、3、481、纯空白、控制字符 | 合法文本 UTF-8 往返一致；非法输入明确拒绝或按已确认服务端规则处理 |
| SJ-BND-08 | 销售地址 | 长度 4、200 | 空、3、201、纯空白 | 新增和修改规则一致，失败后原记录不变 |
| SJ-BND-09 | 经度 | -180、-179.999999、0、179.999999、180 | 小于 -180、大于 180、非数值、空、小数超过六位 | 接受范围内数值并按六位展示；越界不得提交；精度处理可预测 |
| SJ-BND-10 | 纬度 | -90、-89.999999、0、89.999999、90 | 小于 -90、大于 90、非数值、空、小数超过六位 | 接受范围内数值并按六位展示；越界不得提交 |
| SJ-BND-11 | 分页 | 页码 1；页长 1、默认 5；末页和超末页 | 页码 0、负数、非数字；页长 0、负数、极大值 | 合法分页总数和页数一致；非法输入不得导致 500、全表无界返回或资源耗尽 |
| SJ-BND-12 | 图片字节大小 | 有效图片小于 2 MB、恰好 2 MB | 空文件、2 MB 加 1 字节 | 边界内继续做内容解码；超限拒绝且无文件残留 |
| SJ-BND-13 | 图片像素 | 1×1、8000×8000 的可解码 JPG/PNG | 0 维不可解码、任一边 8001、损坏图片 | 合法边界可保存；无效或过大尺寸拒绝 |
| SJ-BND-14 | 图片类型组合 | 扩展名、MIME、真实内容均为 JPG 或 PNG | MIME 合法但内容伪造、扩展名伪装、GIF/PDF/脚本、双扩展名 | 以服务端 MIME 白名单加真实解码为准，不能只信文件名 |

## 8. 异常与恢复设计

| 设计编号 | 异常注入或触发 | 预期系统行为 | 数据核对 | 恢复验证 |
|---|---|---|---|---|
| SJ-ERR-01 | MySQL 不可用或请求中途断连 | 健康状态非 UP；业务请求给出统一错误，不泄漏连接信息 | 新增/修改事务无部分提交 | 恢复连接后健康、登录和核心查询通过 |
| SJ-ERR-02 | Redis 不可用、令牌键丢失或过期 | 新登录/鉴权失败行为明确；不得退化为相信客户端角色 | 业务数据不因鉴权异常被修改 | Redis 恢复后重新登录可用，旧无效令牌不复活 |
| SJ-ERR-03 | 上传目录只读、空间不足或卷未挂载 | 上传返回统一失败，页面保留可重试状态，不返回虚假 URL | 药品记录不应绑定不存在的新图片 | 修复目录/卷后重新上传和读取成功 |
| SJ-ERR-04 | 上传完成但药品更新失败 | 页面提示失败并可再次操作 | 药品仍保留旧图片；新文件列入隔离清理清单 | 重试成功后仅绑定一个目标 URL，孤立文件可追踪 |
| SJ-ERR-05 | 新增医生第二步失败 | 整个事务回滚 | 不存在孤立账号或医生记录 | 修复后重新提交仅生成一组一致记录 |
| SJ-ERR-06 | 药品和销售地点关系写入中途失败 | 药品及关系整体回滚 | 无部分 `drug_sale`、无重复关联或孤儿关系 | 修复后新增/修改、查询和删除完整通过 |
| SJ-ERR-07 | 删除被政策引用的公司/城市 | 返回冲突/业务拒绝，不静默级联 | 主记录和全部引用仍存在 | 解除引用后再删除，结果与约定一致 |
| SJ-ERR-08 | 请求超时、断网、快速重复点击提交 | 页面不应无限加载或重复创建；恢复后可查询真实状态 | 以唯一运行标识检查最多一条有效记录，重复项视为缺陷 | 网络恢复后刷新，列表与数据库一致 |
| SJ-ERR-09 | 浏览器 localStorage 读写抛错或值为乱码 | 应用保持可用；主题回到默认或仅当前会话切换 | 不影响 Token 之外的业务数据；不得清空表单 | 恢复存储后可正常持久化 `classic`/`ios26` |
| SJ-ERR-10 | 首页图片 404、加载慢或资源被缓存 | 核心数量、图表、导航仍可用；布局不塌陷 | 不产生业务写入 | 资源恢复或缓存更新后图片显示，主题切换仍可用 |
| SJ-ERR-11 | Docker 仍使用旧前端层或浏览器缓存旧资源 | 测试门禁识别版本不一致并停止功能结论 | 不以旧页面结果签署新提交 | 强制重建镜像并刷新缓存后重新执行烟测和受影响用例 |
| SJ-ERR-12 | 后端容器重启或 Compose 停止后重启 | 健康检查经历非 UP 后恢复；前端给出可恢复错误 | MySQL 数据不丢失；命名卷图片仍存在 | 健康、登录、主题、列表和已上传图片再次通过烟测 |
| SJ-ERR-13 | 响应出现 401、403、404、409、500 或业务失败码 | 前端提示与状态相符，不把失败当成功，不出现未捕获异常 | 失败写请求均核对数据不变 | 修复后按同状态码类别和关联模块回归 |

## 9. 数据设计

| 数据组 | 数据构造 | 使用场景 | 隔离标识 | 预期清理 |
|---|---|---|---|---|

| SJ-DATA-07 坐标 | 经度 -180/0/180，纬度 -90/0/90，六位小数及越界值 | 销售地点边界和展示 | 临时销售地点名称 | 删除销售地点前先清理药品关系 |
| SJ-DATA-08 上传文件 | 合法 JPG/PNG、空文件、伪造内容、2 MB 边界、8000/8001 像素 | 上传安全、公开读取、卷持久化 | 文件哈希、返回 UUID 文件名、运行清单 | 业务记录走接口删除；隔离卷文件按批准清单清理 |
| SJ-DATA-09 令牌 | 管理员、医生、伪造、过期、退出后、裸格式和 Bearer 格式 | 认证和权限决策表 | 报告只记录令牌类别和脱敏摘要 | 退出并清除本地测试态，不保存完整令牌 |
| SJ-DATA-10 中文编码 | 中文姓名、医院、公司、城市地址、长政策文本、中英文数字混合 | 请求、响应、数据库、页面和报告编码 | 使用 UTF-8 且文件名为中文 | 清理后检查报告与数据库无替换字符或乱码 |
