## 四、关键工作与成果

1. **42 个唯一 API**：覆盖城市、公司、销售地点、公司政策、医保政策、医生、药品、必备材料 8 类业务 CRUD + 认证/权限/仪表盘/上传/健康检查。
2. **httpOnly Cookie 持久登录**：
   - 登录成功下发 httpOnly + Secure + SameSite=Lax cookie，Max-Age 7 天，响应体不再返回 token；
   - `TokenAuthenticationFilter` 优先从 cookie 读取 token，兼容原 Authorization 头；
   - `TokenService` 命中会话时滑动续期 Redis TTL；
   - 新增 `GET /api/session` 供前端无登录态时探测恢复；
   - 登出清除 cookie 并删除 Redis 会话。
3. **服务端鉴权**：管理员（ROLE_1）全量；医生（ROLE_2）仅业务 GET，写接口由 `@PreAuthorize` 拒绝；角色取自服务端会话，不信任客户端 roleName。
4. **会话安全**：Token 为随机不透明令牌，Redis key 存 SHA-256 摘要，不存明文；密码 BCrypt。
5. **事务边界**：医生新增/删除、密码重置（更新密码 + 失效旧 Token + 写审计）、药品 + drug_sale 关系保存均为单事务，失败整体回滚。
6. **引用完整性**：公司/城市被政策引用时拒绝删除（RESTRICT），不静默级联。
7. **上传安全**：2MB 上限、仅 JPEG/PNG、图片解码校验真实内容、UUID 文件名、可配置目录与公开基址。
8. **仪表盘接口**：`GET /api/dashboard` 一次返回 8 类计数 + 医生职称 + 治疗类型 + 最新资讯。

## 六、关键设计要点

- **不把权限交给客户端**：前端零 token，登录态仅服务端可恢复；roleName 不可信。
- **滑动续期**：活跃用户的会话 TTL 自动刷新，闲置 7 天过期。
- **配置化 cookie**：cookie 参数经 `CookieProperties` 绑定 `app.auth.cookie.*`，compose 注入 `TOKEN_TTL=7d` 与后端默认对齐。
- **JDK 17 门禁**：`release=17` 编译 + Temurin 17 实际启动作为交付门禁，不使用 Lombok 降低旧 Spring Boot 与新 JDK 组合风险。
