# 系统开发工程师

> 本文档由系统（全栈）开发工程师视角，基于 Git 提交历史重新撰写，记录慧医数字医疗应用系统从"仅有前端、后端缺失"到"可运行前后端系统"的完整开发历程。覆盖需求、架构、数据库、前后端、测试、部署、CI/CD、UML 与文档全链路。

## 一、角色职责

负责系统整体架构与技术选型、前后端契约与联调、数据库迁移、部署拓扑、CI/CD 流水线、双仓库同步、UML 设计文档与过程证据归档，保证系统可构建、可部署、可验证、可追溯。

## 二、系统目标

基于现有 Vue 前端、需求规格说明书与 `sql/medical.sql`，补齐 Spring Boot 后端，连接远程 MySQL/Redis，完成数据库导入、接口测试、前后端联调、部署验证，并保留可复核的过程文档与证据。

## 三、技术栈全景

| 层 | 选型 |
|---|---|
| 前端 | Vue 3.5 + Vuex 4 + vue-router 4 + Vite 6 + Element Plus + Echarts 4.9 + 高德地图 |
| 后端 | Spring Boot 2.5.3 + Spring Security + MyBatis（JDK 17） |
| 数据库 | MySQL 8（medicine schema，20 表） |

| 部署 | Docker Compose（nginx:alpine + Temurin JRE 17） |

| 版本控制 | Git 双仓库（GitHub origin/main + 华为云 codearts/master） |
| 设计文档 | PlantUML（18 张 UML 图） |
